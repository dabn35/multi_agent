# S.E.E. v2 프로젝트 규칙

이 파일은 GitHub Copilot, Gemini CLI, Antigravity에서 공통으로 참고할 프로젝트 규칙입니다.

## 프로젝트

- **이름**: S.E.E. v2 (Skin & Eating Habits Experience)
- **목표**: 식습관과 피부 상태의 상관관계를 추적하고 화장품을 추천하는 에이전트
- **기본 실행**: `streamlit run see_v2.py`
- **기본 작업 파일**: `see_v2.py`
- **주요 출력**: `data/daily_records.json`, Streamlit 대시보드

## 에이전트 역할

1. **데이터 수집 에이전트**: 사용자 입력(식단, 피부 상태) 처리
2. **분석 에이전트**: 7일 통계, 음식/증상 패턴 분석, 요약 리포트 생성
3. **추천 에이전트**: 피부 상태 기반 규칙 매칭, 화장품 솔루션 제시

## 작업 방식

1. 파일을 수정하기 전에 먼저 계획을 제시한다.
2. 한 번에 많은 파일을 바꾸지 않는다.
3. 기존 구조를 유지하면서 작은 단위로 고친다.
4. 오류가 나면 새 기능을 추가하지 말고 오류만 고친다.
5. 변경 후 실행 명령과 확인 결과를 보고한다.

## 금지

- 불필요한 패키지 설치 (streamlit, pandas만 사용)
- API 키 하드코딩
- 수업 범위를 벗어난 복잡한 프레임워크 추가
- Playwright MCP 연결
- 사용자가 요청하지 않은 대규모 리팩터링

## Streamlit 개발 규칙

- `streamlit run` 명령으로 즉시 실행 가능한 코드
- 토글 버튼: CSS 스타일링 + `st.session_state` 활용
- 데이터 저장: 클릭 시 JSON 파일에 자동 저장
- 상태 관리: streamlit session_state를 기본으로 사용
