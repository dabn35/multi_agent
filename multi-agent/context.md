# skin care tool 프로젝트 컨텍스트

프로젝트 경로: C:\agenticai\multi-agent

## 목표

사용자의 간단한 일기형 기록(예: "오늘 피부가 좀 붉네?")을 입력받아, 자동으로 환경·위치·시간 정보를 태깅하고 제품 사용 기록과 결합해 장기적으로 피부의 인과관계를 분석하고 예측 기반 처방을 제안합니다.

## 입력

- 사용자가 남긴 단문 텍스트(자유형). 예: "오늘 얼굴이 따끔해요".
- 사용자가 수동으로 추가한 사용 제품 정보(선택적).

## 자동 수집(에이전트/API)

- 위치 기반 기상·대기(온도, 습도, 미세먼지, 자외선 등) 정보
- 시간·날씨 예측(오늘/내일) 데이터

## 출력

- 하루 단위 레코드: `사용자 상태 + 사용 제품 + 환경 태그`
- 누적 분석 리포트: 인과관계 히트맵, 상관/회귀 결과 요약
- 예측 처방: 오늘의 예방 루틴 권장사항과 알림 메시지

## 에이전트 역할(요약)

1. 입력 파서 에이전트: 사용자의 자연어 입력에서 간단한 상태 레이블(예: 발적, 트러블, 건조 등)을 추출합니다.
2. 컨텍스트 태깅 에이전트(API): 위치/시간으로 외부 API를 호출해 환경 태그를 자동으로 생성합니다.
3. 저장 에이전트: 태그된 레코드를 DB/파일에 적재합니다.
4. 분석 에이전트: 누적 데이터를 바탕으로 환경 vs 제품 인과 가능성(통계·간단한 인과추정)을 도출합니다.
5. 처방/알림 에이전트: 예측 기반 예방 루틴을 제안하고 알림을 생성합니다.
6. 검토자 에이전트: 데이터 부족·편향 가능성·신뢰구간을 계산해 사용자에게 불확실성을 설명합니다.

## 제약 및 운영 원칙

- 개인 민감 데이터는 암호화/로컬 저장을 기본으로 하며, 외부 전송 시 반드시 사용자의 동의가 필요합니다.
- API 키는 `.env`에 보관하고 리포지토리에는 포함하지 않습니다.
- 초기 PoC(증명) 단계에서는 외부 패키지 추가를 최소화합니다.

## 실행(개발)

개발 시 샘플 데이터를 준비하고 `run_agents.py`로 실행합니다. 자세한 커맨드는 `README.md`에 적습니다.

## 피부 상태 측정 매커니즘 (추가)

- **시각적 정량화**: 사용자가 촬영한 얼굴 사진을 통해 홍조도(RGB 기반), 트러블 개수/면적, 모공 상태, 유분(광택) 등 객관적 지표를 추출합니다. 촬영 메타데이터(기온, 습도 등)와 결합해 환경 영향 분석을 지원합니다.
- **주관적 피드백**: 슬라이더(1~5점)를 통해 따가움/가려움/속당김 등 주관적 감각을 수집하고, 이미지 기반 지표와 상호 검증합니다.
- **맥락 태그**: 생리, 수면, 음주, 스트레스, 운동 여부 등 빠른 버튼형 태그로 수집하여 추가 공변량으로 사용합니다.

## 데이터 구조(요약)

레코드는 하루 단위 이벤트로 저장됩니다. 핵심 필드:

- `environment`(X1): `temperature`, `humidity`, `pm25`, `uv_index`
- `exposures`(X2): `products_used`, `cleanses_per_day`, `mask_used`
- `internal`(X3): `sleep_hours`, `menstrual_phase`, `stress_level`, `exercise`
- `visual_metrics`(Y_obj): `redness_score`, `lesion_count`, `lesion_area`, `pore_score`, `shine_score`
- `subjective_scores`(Y_subj): 슬라이더 응답
- `meta`: `user_id`, `timestamp`, `photo_uri`, `location`

이 데이터 구조는 분석 파이프라인(상관분석, 회귀, 간단한 인과추정)에 바로 투입될 수 있도록 설계되었습니다.
