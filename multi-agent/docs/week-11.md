# Week 11. 쉬운 멀티에이전트 뼈대 만들기

> 10주차에 익힌 코딩에이전트 도구를 사용하여 일정공지 에이전트의 첫 번째 버전을 만든다.  
> 이번 주에는 LangGraph Supervisor를 쓰지 않는다. Python 함수 3개를 에이전트 역할로 나누어 가장 쉬운 멀티에이전트 구조를 만든다.

---

## 학습 목표

- 9주차 역할 분리표를 코드 구조로 바꾼다.
- 일정공지 에이전트의 최소 동작 흐름을 만든다.
- 코딩에이전트에게 “계획 먼저, 코드 나중” 방식으로 작업을 맡긴다.
- 함수 3개가 각각 다른 에이전트 역할을 하도록 구성한다.
- 실행 결과를 보고 다음 주 개선 항목을 `todo.md`에 남긴다.

---

## 이번 주 핵심 원칙

1. 작게 만든다. 파일 하나에서 시작한다.
2. LangGraph, RAG, 데이터베이스는 쓰지 않는다.
3. 코딩에이전트가 코드를 만들게 하되, 사람이 흐름을 검토한다.
4. 중간 결과를 모두 출력한다.
5. 완벽한 추출보다 “역할이 나뉘어 실행되는 것”을 우선한다.

## 11.2 오늘 만드는 것

오늘의 산출물은 `schedule_agent.py`이다. 이 파일은 일정공지 에이전트의 v0이다.

```text
[sample notices]
      ↓
extract_facts()
      ↓
classify_schedule()
      ↓
write_guide()
      ↓
print result
```

각 함수는 하나의 에이전트 역할을 맡는다.

| 함수 | 에이전트 역할 | 하는 일 |
|---|---|---|
| `extract_facts` | 사실 추출 에이전트 | 공지에서 제목, 대상, 날짜, 준비물을 찾는다 |
| `classify_schedule` | 일정 분류 에이전트 | 공지를 사용자 유형별로 나눈다 |
| `write_guide` | 안내문 작성 에이전트 | 사용자가 해야 할 일을 짧게 쓴다 |

이번 주에는 정교한 자연어 처리를 목표로 하지 않는다. 문자열 규칙과 간단한 리스트/딕셔너리로 충분하다.

## 11.4 코딩에이전트에게 줄 첫 요청

10주차에 선택한 주 도구를 사용한다. Copilot, Gemini CLI, Antigravity 중 무엇을 써도 된다. 다만 요청 방식은 같다.

먼저 계획만 받는다.

```text
context.md와 todo.md를 읽어줘.
오늘은 schedule_agent.py 하나만 만들 거야.
LangGraph나 외부 패키지는 쓰지 말고 Python 기본 코드로만 작성할 거야.

아직 파일을 만들지 말고, 다음 구조로 만들 계획만 제시해줘.
1. sample notices
2. extract_facts 함수
3. classify_schedule 함수
4. write_guide 함수
5. main 함수
```

계획을 확인한 뒤 코드 생성을 요청한다.

```text
좋아. 이제 schedule_agent.py를 만들어줘.
코드는 초보자가 읽을 수 있게 단순하게 작성해줘.
각 함수가 하는 일이 다르게 보이도록 하고, main에서 각 단계의 중간 결과를 print로 보여줘.
외부 패키지는 사용하지 마.
```

## 11.5 사람이 확인할 핵심 코드

코딩에이전트가 만든 코드 전체를 완벽히 이해하려고 하지 않는다. 오늘은 세 부분만 확인한다.

### 확인 1. 함수가 3개로 나뉘었는가

```python
def extract_facts(notices_text):
    ...

def classify_schedule(facts):
    ...

def write_guide(grouped):
    ...
```

함수 이름은 달라도 된다. 그러나 역할은 나뉘어야 한다.

### 확인 2. 앞 단계 출력이 다음 단계 입력으로 들어가는가

```python
facts = extract_facts(SAMPLE_NOTICES)
grouped = classify_schedule(facts)
guide = write_guide(grouped)
```

이 흐름이 보이면 최소 멀티에이전트 뼈대는 성공이다.

### 확인 3. 중간 결과가 출력되는가

```python
print("=== facts ===")
print(facts)
```

중간 결과가 보여야 다음 주에 수정하기 쉽다.

## 11.6 실행하기

터미널에서 실행한다.

```bash
python schedule_agent.py
```

실행이 안 되면 코딩에이전트에게 오류 메시지를 그대로 준다.

```text
다음 오류가 나왔어.

(여기에 오류 메시지 붙여넣기)

schedule_agent.py의 구조는 유지하고, 오류만 고쳐줘.
수정한 이유를 짧게 설명해줘.
```

## 11.7 좋은 결과의 기준

이번 주 결과는 예쁘지 않아도 된다. 다음 조건을 만족하면 충분하다.

| 기준 | 통과 조건 |
|---|---|
| 실행 | `python schedule_agent.py`가 실행된다 |
| 역할 분리 | 함수가 최소 3개로 나뉘어 있다 |
| 흐름 | 사실 추출 → 분류 → 안내문 작성 순서가 보인다 |
| 중간 출력 | 각 단계 결과가 터미널에 나온다 |
| 수정 가능성 | 다음 주에 사실 추출 부분을 고칠 수 있다 |
