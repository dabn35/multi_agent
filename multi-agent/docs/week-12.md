# Week 12. 일정공지 에이전트 1차 구현: 사실 추출과 일정 분류

> 11주차에 만든 `schedule_agent.py`를 개선한다.  
> 이번 주 목표는 공지에서 핵심 정보를 뽑고, 그 결과를 Markdown 표로 저장하는 것이다.

---

## 학습 목표

- 공지 텍스트에서 제목, 대상, 날짜, 해야 할 일, 준비물을 추출한다.
- 추출 결과를 리스트와 딕셔너리 형태로 정리한다.
- 일정공지 에이전트의 “사실 추출자”와 “일정 분류자” 역할을 개선한다.
- 결과를 `output.md`에 Markdown 표로 저장한다.
- 코딩에이전트에게 작은 단위로 수정 요청하는 방법을 익힌다.

---

## 이번 주 핵심 원칙

1. 새 기능은 하나씩 추가한다.
2. 코드 전체를 다시 만들게 하지 않는다.
3. 11주차 구조를 유지한 채 `extract_facts`와 `classify_schedule`만 개선한다.
4. 완벽한 자연어 처리를 목표로 하지 않는다.
5. 실행 결과 파일을 남긴다.

---

## 12.1 지난주 코드 점검

수업 시작 전에 다음 명령이 실행되어야 한다.

```bash
python schedule_agent.py
```

실행이 안 되면 오늘 새 기능을 추가하지 않는다. 먼저 실행 오류를 고친다.

코딩에이전트에게 이렇게 요청한다.

```text
schedule_agent.py가 실행되지 않아.
아래 오류만 고쳐줘.
함수 구조는 유지하고, 새 기능은 추가하지 마.

(오류 메시지)
```

## 12.2 오늘의 목표 출력

오늘은 최종 결과를 `output.md`에 저장한다.

```markdown
| 제목 | 대상 | 날짜 | 해야 할 일 | 준비물 |
|---|---|---|---|---|
| 수강신청 안내 | 전 학년 | 2026-02-10 ~ 02-12 | 수강신청 | 포털 로그인, 학과 시간표 |
| 졸업논문 제출 안내 | 4학년 | 2026-05-31 | 졸업논문 제출 | PDF 파일, 학과 이메일 |
| 교환학생 설명회 | 2학년 이상 | 2026-04-03 15:00 | 설명회 참석 | 장소 확인 |
```

표가 정확히 이와 같을 필요는 없다. 다만 제목, 대상, 날짜, 해야 할 일, 준비물 열이 있어야 한다.

---

## 12.3 데이터 구조 정하기

사실 추출 결과는 다음처럼 단순한 딕셔너리 리스트로 둔다.

```python
[
    {
        "title": "수강신청 안내",
        "target": "전 학년",
        "date": "2026-02-10 ~ 02-12",
        "task": "수강신청",
        "items": "포털 로그인, 학과 시간표",
    }
]
```

핵심은 하나이다. 다음 단계 에이전트가 읽을 수 있는 형태로 결과를 정리해야 한다.

---

## 12.4 코딩에이전트에게 줄 요청 1: 사실 추출 개선

먼저 `extract_facts`만 고친다.

```text
schedule_agent.py를 읽어줘.
이번에는 extract_facts 함수만 개선할 거야.

목표:
- 공지별로 title, target, date, task, items를 뽑는다.
- 결과는 dict의 list로 반환한다.
- 외부 패키지는 쓰지 않는다.
- classify_schedule, write_guide의 큰 구조는 아직 바꾸지 않는다.

먼저 수정 계획만 말해줘. 아직 파일을 수정하지 마.
```

계획을 확인한 뒤 실행한다.

```text
좋아. 이제 extract_facts 함수만 수정해줘.
수정 후 main에서 facts를 보기 좋게 출력해줘.
```

## 12.5 코딩에이전트에게 줄 요청 2: Markdown 표 저장

다음으로 `output.md` 저장 기능을 추가한다.

```text
schedule_agent.py에 save_markdown_table 함수를 추가해줘.

요구사항:
- 입력은 facts 리스트로 받는다.
- output.md 파일에 Markdown 표를 저장한다.
- 표 열은 제목, 대상, 날짜, 해야 할 일, 준비물이다.
- pathlib.Path를 사용한다.
- 기존 함수 구조는 유지한다.
```

코드는 길 필요가 없다. 핵심은 결과 파일이 남는 것이다.

## 12.6 일정 분류 에이전트 개선

분류는 처음에는 단순한 규칙으로 충분하다.

| 조건 | 사용자 유형 |
|---|---|
| 대상에 `전 학년` 포함 | 전체 학생 |
| 대상에 `1학년` 포함 | 1학년 |
| 대상에 `2학년` 포함 | 2학년 |
| 대상에 `4학년` 포함 | 4학년 |
| 제목에 `교환학생` 포함 | 교환학생 준비생 |

코딩에이전트 요청:

```text
classify_schedule 함수를 개선해줘.

규칙:
- facts 리스트를 입력으로 받는다.
- 사용자 유형별로 공지를 묶은 dict를 반환한다.
- 사용자 유형은 전체 학생, 1학년, 2학년, 4학년, 교환학생 준비생 정도만 사용한다.
- 복잡한 AI 판단은 하지 말고 문자열 포함 여부로 분류한다.
```
