# Week 13. 일정공지 에이전트 개선: 사용자별 안내문과 검토자

> 12주차에는 공지에서 일정 정보를 추출하고 `output.md` 표를 만들었다.  
> 이번 주에는 사용자 유형별 안내문을 만들고, 검토자 에이전트가 빠진 정보와 위험한 표현을 확인하게 한다.

---

## 학습 목표

- 사용자 유형별 안내문을 생성한다.
- 안내문 작성 에이전트와 검토자 에이전트의 역할을 구분한다.
- 코딩에이전트에게 “기존 구조를 유지한 채 기능 추가”를 요청한다.
- 최종 출력 파일 `output_user_guide.md`를 만든다.
- 결과의 한계와 수정 필요 사항을 기록한다.

---

## 이번 주 핵심 원칙

1. 12주차 코드를 버리지 않는다.
2. 안내문 작성 기능을 추가하되, 추출과 분류 구조는 유지한다.
3. 검토자 에이전트는 복잡한 평가자가 아니라 체크리스트 역할만 한다.
4. 사용자가 오해할 수 있는 문장은 줄인다.
5. 최종 제출을 생각하며 README와 실행 흐름을 정리하기 시작한다.

---

## 13.1 오늘의 구조

이번 주가 끝나면 에이전트 구조는 다음과 같다.

```text
[공지 입력]
   ↓
[사실 추출 에이전트]
   ↓
[일정 분류 에이전트]
   ↓
[사용자별 안내문 작성 에이전트]
   ↓
[검토자 에이전트]
   ↓
[output_user_guide.md]
```

함수 이름으로 보면 다음과 같다.

| 함수 | 역할 |
|---|---|
| `extract_facts` | 공지에서 사실을 뽑는다 |
| `classify_schedule` | 사용자 유형별로 공지를 묶는다 |
| `write_user_guides` | 사용자별 안내문을 쓴다 |
| `review_guides` | 날짜 누락, 대상 누락, 과장 표현을 확인한다 |
| `save_user_guides` | 결과를 파일로 저장한다 |

---

## 13.2 사용자 유형 정하기

처음에는 사용자 유형을 고정한다.

```text
전체 학생
1학년
2학년
4학년
교환학생 준비생
```

모든 유형을 반드시 만들 필요는 없다. 입력 공지에 따라 일부 유형만 나와도 된다.

---

## 13.3 안내문 출력 예시

`output_user_guide.md`는 다음과 같은 형태면 된다.

```markdown
# 사용자별 일정 안내

## 4학년

### 해야 할 일
- 2026-05-31까지 졸업논문 PDF 파일을 학과 이메일로 제출한다.
- 수강신청 기간에 졸업요건을 함께 확인한다.

### 주의할 점
- 제출 형식이 PDF인지 확인한다.
- 마감 이후 제출 가능 여부는 공지에 없으므로 학과에 확인해야 한다.

## 교환학생 준비생

### 해야 할 일
- 2026-04-03 15:00에 교환학생 설명회에 참석한다.

### 주의할 점
- 장소는 국제교류센터이다.
```

## 13.4 코딩에이전트 요청 1: 안내문 작성 함수 추가

먼저 계획을 받는다.

```text
schedule_agent.py를 읽어줘.
12주차 구조를 유지하면서 사용자별 안내문 작성 기능을 추가하려고 해.

추가할 함수:
- write_user_guides(grouped)
- save_user_guides(guides, path="output_user_guide.md")

아직 파일을 수정하지 말고, 어떤 데이터를 입력받아 어떤 문자열을 만들지 계획만 말해줘.
```

계획이 적절하면 수정시킨다.

```text
좋아. 이제 write_user_guides와 save_user_guides를 추가해줘.

조건:
- grouped dict를 입력으로 받는다.
- 사용자 유형별로 해야 할 일과 주의할 점을 Markdown으로 만든다.
- output_user_guide.md에 저장한다.
- 기존 extract_facts, classify_schedule 함수의 큰 구조는 유지한다.
```

## 13.5 코딩에이전트 요청 2: 검토자 에이전트 추가

검토자는 안내문을 다시 쓰는 역할이 아니다. 빠진 정보와 위험한 표현을 알려주는 역할이다.

검토 기준:

| 기준 | 확인 내용 |
|---|---|
| 날짜 | 날짜 또는 마감일이 빠진 안내가 있는가 |
| 대상 | 누구에게 해당하는지 불분명한가 |
| 근거 | 공지에 없는 내용을 단정했는가 |
| 행동 | 사용자가 무엇을 해야 하는지 보이는가 |

요청문:

```text
review_guides 함수를 추가해줘.

역할:
- guides 문자열을 입력으로 받는다.
- 날짜가 없는 항목, 대상이 불분명한 항목, 공지에 없는 단정 표현을 점검한다.
- 결과는 review_report 문자열로 반환한다.
- 복잡한 AI 판단은 하지 말고 간단한 체크리스트 문장으로 작성한다.
- main에서 검토 결과를 출력하고 review_report.md에도 저장한다.
```
