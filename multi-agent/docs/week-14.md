# Week 14. 일정공지 에이전트 최종 정리와 제출 준비

> 이번 주에는 새 개념을 배우지 않는다.  
> 11~13주차에 만든 일정공지 에이전트를 실행 가능한 상태로 정리하고, 최종 제출물과 발표 자료를 완성한다.

---

## 학습 목표

- 최종 제출 기준에 맞게 파일을 정리한다.
- 다른 사람이 README만 보고 실행할 수 있게 만든다.
- 실행 결과와 한계를 설명할 수 있게 준비한다.
- 코딩에이전트를 사용해 오류 수정, 문서 정리, 발표자료 초안을 만든다.
- 15주차 발표를 위한 1분 데모 흐름을 만든다.

---

## 이번 주 핵심 원칙

1. 새 기능을 크게 추가하지 않는다.
2. 실행이 안 되는 코드를 제출하지 않는다.
3. README, 출력 예시, 도구 사용 기록을 반드시 남긴다.
4. 코딩에이전트가 만든 설명이 실제 코드와 맞는지 확인한다.
5. 발표는 “무엇을 만들었는가, 어떻게 나누었는가, 어디까지 되는가”를 중심으로 한다.

---

## 14.1 최종 제출 파일

최종 제출 폴더에는 다음 파일이 있어야 한다.

```text
schedule_agent.py
sample_notices.txt
output.md
output_user_guide.md
review_report.md
README.md
context.md
todo.md
.github/copilot-instructions.md
tool_usage_log.md
presentation.md
```

파일별 역할은 다음과 같다.

| 파일 | 설명 |
|---|---|
| `schedule_agent.py` | 최종 실행 코드 |
| `sample_notices.txt` | 입력 공지 예시 |
| `output.md` | 공지별 일정 표 |
| `output_user_guide.md` | 사용자별 안내문 |
| `review_report.md` | 검토자 에이전트의 점검 결과 |
| `README.md` | 실행 방법과 프로젝트 설명 |
| `context.md` | 프로젝트 목표와 제약 |
| `todo.md` | 개발 과정과 남은 일 |
| `.github/copilot-instructions.md` | Copilot 작업 규칙 |
| `tool_usage_log.md` | 코딩에이전트 사용 기록 |
| `presentation.md` | 발표자료 초안 |

---

## 14.2 실행 점검

먼저 실행한다.

```bash
python schedule_agent.py
```

실행 후 다음 파일이 생성되거나 갱신되는지 확인한다.

```text
output.md
output_user_guide.md
review_report.md
```

실행 오류가 있으면 새 기능을 추가하지 말고 오류만 고친다.

코딩에이전트 요청:

```text
schedule_agent.py를 실행했더니 아래 오류가 나왔어.
새 기능을 추가하지 말고, 실행 오류만 고쳐줘.
기존 함수 이름과 전체 흐름은 유지해줘.

(오류 메시지)
```

## 14.3 README 최종 정리

README에는 다음 항목이 있어야 한다.

```markdown
# 일정공지 에이전트

## 한 문장 설명

## 실행 방법

## 입력 파일

## 출력 파일

## 에이전트 역할
| 역할 | 함수 | 설명 |
|---|---|---|

## 사용한 코딩에이전트

## 현재 한계
```

코딩에이전트 요청:

```text
현재 파일을 기준으로 README.md를 최종 정리해줘.
실제로 구현되지 않은 기능은 쓰지 마.
실행 방법은 python schedule_agent.py 하나로 설명해줘.
에이전트 역할은 함수 이름과 연결해서 표로 정리해줘.
현재 한계도 솔직하게 써줘.
```

## 14.5 발표자료 만들기

발표자료는 길 필요가 없다. `presentation.md`에 다음 순서로 작성한다.

```markdown
# 일정공지 에이전트 발표

## 1. 문제
여러 공지를 읽고 날짜와 해야 할 일을 놓치기 쉽다.

## 2. 만든 것
공지에서 일정 정보를 추출하고, 사용자별 안내문을 만든다.

## 3. 멀티에이전트 구조
사실 추출자 → 일정 분류자 → 안내문 작성자 → 검토자

## 4. 실행 결과
output.md, output_user_guide.md, review_report.md

## 5. 사용한 코딩에이전트
Copilot / Gemini CLI / Antigravity 사용 내용

## 6. 한계
정교한 자연어 이해는 부족하다. 입력 형식이 크게 바뀌면 오류가 날 수 있다.
```

코딩에이전트 요청:

```text
README.md와 출력 파일을 바탕으로 presentation.md 초안을 만들어줘.
발표 시간은 3분이다.
문장은 짧게 쓰고, 실제 구현된 내용만 포함해줘.
```

## 14.6 1분 데모 흐름

발표 전에 다음 흐름을 1분 안에 보여줄 수 있어야 한다.

```text
1. sample_notices.txt를 보여준다.
2. python schedule_agent.py를 실행한다.
3. output.md 표를 보여준다.
4. output_user_guide.md 안내문을 보여준다.
5. review_report.md 검토 결과를 보여준다.
```

데모 중 설명할 문장:

```text
이 프로젝트는 공지 텍스트를 입력으로 받아 사실 추출, 일정 분류, 안내문 작성, 검토의 네 역할로 나누어 처리합니다.
```
