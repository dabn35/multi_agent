## 내 에이전트 주제
- 에이전트 이름: skin care tool
- 해결하려는 문제: 마케팅 문구에 의존한 화장품 선택 대신, 개인의 환경적·생체적 데이터 기반 인과관계 분석과 맞춤 솔루션 제공
- 입력:
  - 위치 기반 기상 정보 (기온, 습도, 미세먼지, 자외선 지수)
  - 사용자 자연어 한 줄 상태 기록
  - 시각 데이터(피부 근접 사진, 옵션)
- 출력:
  - 데일리 레포트: 오늘의 피부 환경 지수 및 추천 루틴
  - 인사이트 리포트(주/월): 득/독 성분 분석표
  - 예측 알림(선제적 처방)

## 일정공지 예시에서 바꾼 점
| 일정공지 예시 | 내 에이전트 |
|---|---|
| 공지문 | 단일 공지문이 아닌 사용자 개인의 자연어 기록을 입력으로 사용 |
| 날짜/대상/준비물 추출 | 날짜·대상 대신 `환경(기상)`·`제품 사용 여부`·`주관적 상태` 추출 |
| 학년별 분류 | 학년 분류 대신 `환경 클러스터(예: 고온다습, 저온건조 등)` 분류 도입 |
| 사용자별 안내문 | 학년별 안내가 아니라 `개인별 데일리 루틴·성분 권고` 생성 |

## 코딩에이전트가 확인한 항목
| 확인 항목 | 통과 여부 | 근거 |
|---|---:|---|
| 함수가 3개 역할로 나뉘었는가 | 통과 | `extract_facts`, `classify_items`, `write_output` 함수가 `my_agent.py`에 정의됨 |
| 앞 단계 출력이 다음 단계 입력으로 들어가는가 | 통과 | `rec = extract_facts(ln)` → `records.append(rec)` → `analysis = classify_items(records)` → `report = write_output(records, analysis)` 호출 흐름 존재 |
| 중간 결과가 출력되는가 | 통과 | `print('\n=== 1) extract_facts 결과 ===')`, 각 `print(rec)`, `print('env_clusters:')`, `print('=== 3) write_output 결과 ===')` 등 출력 구문 존재 |
