## 강의 목표

본 과목을 이수하면 다음 역량을 갖추게 된다.

1. AI 협업 개발의 역할 분담(설계·검증·책임)을 이해하고, 검증 가능한 산출물을 만들 수 있다
2. MCP(Model Context Protocol)의 아키텍처를 이해하고, 도구·리소스를 설계·구현할 수 있다
3. 외부 API를 MCP 서버로 래핑하고, OAuth 2.1 및 A2A 프로토콜의 개념을 설명할 수 있다
4. LangChain과 LangGraph로 도구를 활용하는 에이전트와 상태 기반 워크플로우를 구현할 수 있다
5. 멀티에이전트 시스템(Supervisor·Handoff·평가)을 설계·구현하고, 단일 에이전트와의 차이를 판단할 수 있다
6. 코딩에이전트 도구를 활용하여 작은 멀티에이전트 프로젝트를 설계·구현·검증하고, 그 과정을 문서화할 수 있다

## 평가 방식

| 항목 | 비중 | 내용 |
|------|:----:|------|
| 출석 및 참여 | 10% | 15주 중 결석 3회 이상 시 F |
| 주차별 과제 | 30% | 2회차 실습 과제 (매주 제출, 12회) |
| 중간 평가 (미니 프로젝트) | 20% | 7주차 날씨 요약 에이전트로 대체 (기존 중간고사 폐지) |
| 최종 프로젝트 | 40% | 문서화 (5%) + 구현 (20%) + 발표·구두 평가 (15%) — 15주차 발표가 기말고사를 겸함 |

### 주차별 과제 (30%)

| 주차 | 과제 | 제출 형태 |
|:----:|------|----------|

| 14 | 최종 점검본 + 도구 사용 기록 + 발표자료 초안 | GitHub + 발표자료 |

### 최종 프로젝트 (40%) — 기말고사 대체

기말고사를 폐지하고, 15주차 발표를 통해 최종 프로젝트로 통합 평가한다. 발표 후 5분 질의응답이 구두 평가를 겸한다.

**발표 구성** (10분 발표 + 5분 질의응답)
1. 문제 정의와 동기 (2분)
2. 멀티에이전트 구조: 사실 추출자·일정 분류자·안내문 작성자·검토자 (3분)
3. 데모 시연: 입력 공지, 실행 명령, 출력 파일 (3분)
4. 사용한 코딩에이전트와 한계점 (2분)

**평가 기준**

| 항목 | 비중 | 설명 |
|------|:----:|------|
| 작동 여부 | 30% | `python schedule_agent.py` 실행으로 출력 파일이 생성됨 |
| 멀티에이전트 구조 | 25% | 역할이 3개 이상으로 나뉘고 입력·출력이 구분됨 |
| 코딩에이전트 활용 | 20% | Copilot, Gemini CLI, Antigravity 중 최소 2개 사용 기록 |
| 문서화 | 15% | README, `context.md`, `todo.md`, 출력 예시가 제출됨 |
| 발표 및 설명 | 10% | 구조, 데모, 한계를 자신의 말로 설명함 |

**공통 프로젝트**

12~14주차와 최종 발표의 공통 프로젝트는 일정공지 에이전트이다. 개인별 차이는 입력 공지 예시, 사용자 유형, 안내문 표현 수준에서만 둔다. MCP, RAG, LangGraph, 배포는 선택 가산점이며 필수 요구사항이 아니다.
